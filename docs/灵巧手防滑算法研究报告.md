# 灵巧手抓握防滑算法研究报告

**项目**：dexterous-slip-detection  
**目标硬件**：XHAND1 / XHAND1 PRO  
**日期**：2026-07-06  
**状态**：仿真闭环验证完成（番茄酱瓶场景）

---

## 1. 问题描述

### 1.1 核心问题

**如何让灵巧手在抓取与搬运物体时，通过触觉等本体感知信号实时检测相对滑动，并自动增大握力以抑制滑落？**

这是灵巧手从「摆姿势、走轨迹」走向「真抓取、真操作」的关键能力。抓取过程中，物体与手指之间的摩擦力不足、载荷突变或运动加速度过大，都会导致微观滑移乃至宏观滑落；若不能在线感知并及时补偿，再精确的开环轨迹也会在物理层面失败。

### 1.2 业务场景

本研究以**单手握持瓶装物体并抬升搬运**为典型场景：

- 灵巧手（XHAND）已完成 SPIDER 数据集上的抓取轨迹；
- 在轨迹末段追加 **2 s 扩展动作**：模仿末 1 s 手指控制，同时腕部竖直抬升 **10 cm**；
- 若握持力不足，物体在抬升过程中滑落——这正是防滑算法需要介入的时刻。

### 1.3 技术挑战

| 挑战 | 说明 |
|------|------|
| 真机可观测性 | 算法只能使用触觉力、关节本体感知、（可选）视觉/点云等真实传感器，不能依赖仿真器内部的「完美接触滑移速度」等作弊量 |
| 滑动隐蔽性 | 物体可能在**接触点数不变**的情况下发生相对滑动，单纯统计接触数无法可靠检测 |
| 信号噪声 | 接触力瞬时波动大，直接阈值判断易误报或漏报 |
| 响应时效 | 从检测到滑落往往只有数百毫秒窗口，检测与力补偿必须足够快 |
| 仿真到真机 | 仿真中验证的算法需能映射到 XHAND 的 taxel 力阵列与力控接口 |

### 1.4 研究目标

1. 在 MuJoCo 仿真中建立**可复现**的抓取—抬升—滑落测试场景；
2. 在开环轨迹下明确**成功与失败的物理边界**（质量、摩擦等）；
3. 提出并验证**真机可部署**的滑动检测方案；
4. 实现「检测 → 增大握力」闭环，在边界工况上将 **FAIL 恢复为 PASS**。

## 2. 行业研究现状

### 2.1 滑动检测的主流技术路线

业内滑动检测可归纳为五条代表性路线：

| 路线 | 代表工作 | 基本原理 | XHAND1 适配性 |
|------|----------|----------|---------------|
| **① 振动检测** | PapillArray + HDC | 滑动产生特征振动，频谱/形态分类 | ❌ 不适用——XHAND 无振动感知通道 |
| **② 深度学习时空特征** | 压电阵列 + CNN-LSTM | 从触觉阵列时空模式学习滑动特征 | ⚠️ 部分适用，需高采样率时序数据 |
| **③ 物理模型—能量流** | NAIST + Honda（[arXiv:2512.21043](https://arxiv.org/abs/2512.21043)） | 将抓取抽象为「虚拟能量容器」，监测能量守恒偏离 | ✅ 较匹配——仅需接触力 + 接触位置 |
| **④ 端到端强化学习** | DexManip、GR00T-Dexterity | 触觉原始输入直接学习抓取策略 | ✅ 高度适配，但需大量 GPU 训练与数据 |
| **⑤ 视触觉** | GelSight、纬钛 GF515 等 | 高分辨率表面形变 + 光流分析 | ❌ 不适用——XHAND 为电容/电阻式阵列，非视触觉 |

### 2.2 能量流方法（重点参考）

NAIST 与 Honda 研究院提出的**物理能量抽象**方法，将物体视为无质量假设下的「虚拟能量容器」：

- **施加功率** \(P^A\)：各接触点接触力与接触点速度的点积；
- **保留功率** \(\tilde{P}^R\)：无质量假设下应保留在抓取中心的功率；
- **质量估计** \(\tilde{m}\)：由能量平衡反推等效质量；
- **滑动判据**：\(\tilde{m}\) 相对滑动窗口均值显著下降 → 判定滑动。

论文报告在仿真中约 15 次试验、真机约 1 分钟可学会防滑抓取。但该工作**尚未开源**，需自行复现；且完整实现依赖准确的接触点速度，在仿真与真机上均不易稳定获取。

### 2.3 本项目的算法选型考量

| 考量 | 决策 |
|------|------|
| XHAND 无振动通道 | 排除振动检测路线 |
| 当前环境无 GPU | 优先物理可解释、CPU 可运行的方法，RL 作长期路线 |
| 简化版能量流 m̃ 试验 | nominal 工况误报极高（约 297 次/351 接触步），且用手 body 速度近似接触点速度，在「接触数不变但在滑」时失效 |
| 真机部署 | 优先使用触觉力、几何关系、竖直支撑力等**可直接测量**的量 |

在上述约束下，我们提出并验证了两条互补的防滑检测方案（见第 3 节），共用同一握力响应策略。

## 3. 我们提出的解决方案

### 3.1 总体架构

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ 触觉/力学    │ ──▶ │  滑动检测器       │ ──▶ │  握力补偿控制器   │
│ 传感信号     │     │  （方案一 或 二）  │     │  手指闭合 +Δθ   │
└─────────────┘     └──────────────────┘     └─────────────────┘
```

- **检测层**：方案一（几何）或方案二（力学），输出滑动布尔信号；
- **响应层**：检测到滑动后，手指 actuator（索引 6–17）每步增加 **+0.015 rad** 握力，累计上限 **0.25 rad**；
- **启用时机**：仅在轨迹末段的 **extend 段**（2 s 抬升）闭环启用，轨迹回放段保持开环。

### 3.2 方案一：接触中心 vs 物体中心（掌系背离）

**思路**：在手掌坐标系下，比较**力加权接触中心**与**物体中心**的相对运动。即使接触点数不变，物体滑移仍会导致两中心背离。

| 信号 | 仿真实现 | 真机对应 |
|------|----------|----------|
| 接触中心 | \(\sum \|F_i\| p_i / \sum \|F_i\|\) | 触觉 taxel 力 + 位置加权 |
| 物体中心 | body `xpos`（点云质心代理） | 视觉/点云 COM 估计 |
| 参考系 | `right_hand_link` 掌系 | 关节 FK 掌系 |

**判据**（extend 段内）：

1. 记录 extend 起点基准间距 \(sep_{ref}\)；
2. 若 \(|sep - sep_{ref}| \ge 8\) mm → 滑动；
3. 或两中心运动速度差 \(\ge 15\) mm/s → 滑动。

**特点**：对几何相对运动敏感，extend 起始即可触发；不依赖接触数变化。

**代码**：`src/sim/slip_center_detect.py`

### 3.3 方案二：竖直支撑力 \(S(t)\)

**思路**：将手—物接触力投影到**竖直方向（反重力 \(\hat{g}\)）**，度量握持对物体的向上支撑能力。滑动发生时，有效支撑力往往先于物体位置明显变化而下降。

**瞬时支撑力**：

\[
S(t) = \sum_{i \in \text{hand–object}} \max(0,\; \mathbf{f}_i \cdot \hat{\mathbf{g}})
\]

**信号处理管线**：

```
S_raw  →  S_smooth（200 ms 滑动平均）  →  S_avg（2 s 滑动平均）
```

**滑动判据**（`VerticalSupportAntislipDetector`）：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 比值判据 | \(S_{smooth} / S_{avg} < 0.7\) | 支撑力相对近期均值塌缩 |
| 峰值判据 | \(S_{smooth} < 0.95 \times \text{peak}(S_{smooth})\) | extend 段内峰值跟踪，更早预警 |
| Latch | 任一条件触发后保持至 extend 结束 | 防止力短暂回升导致误判「未滑移」而停止加力 |

轨迹阶段持续更新 \(S_{avg}\)（不重置），为 extend 段提供稳定基线。

**特点**：纯力学信号，真机仅需接触力与重力方向；原始 \(S(t)\) 噪声大，需平滑与 latch 配合。

**代码**：`src/sim/slip_vertical_support.py`

### 3.4 方案对比与融合方向

| 维度 | 方案一 | 方案二 |
|------|--------|--------|
| 核心信号 | 几何背离 | 竖直支撑力 |
| 真机依赖 | 触觉 + 物体位姿 | 触觉 + 重力方向 |
| 检测灵敏度 | extend 起始即触发 | 需峰值 latch 才达同等时效 |
| 噪声特性 | 几何量较稳 | 力信号需 200 ms 平滑 |
| 融合建议 | 可作主检测 | 可作力学确认，降低误报 |

当前仿真默认 `--antislip` 接入**方案二**；方案一代码保留，可切换对比。

### 4.2 实验过程

试验分三个阶段，均在**同一开环轨迹**上进行，仅改变物理参数或是否启用防滑。

#### 阶段 A：开环物理摸底

**目的**：在不接入防滑算法的情况下，找到可控的成功/失败边界。

| 组别 | 变体 | 实现 |
|------|------|------|
| 质量组 | ×2、×4、×8、×16、×32 | 缩放物体 body 质量与惯量 |
| 摩擦组 | ÷2、÷4、÷8 | 缩放 hand–object 接触对摩擦系数 |

```bash
python3 scripts/run_ketchup_robustness_sweep.py
```

#### 阶段 B：方案一闭环验证

**目的**：验证掌系中心背离检测 + 握力补偿能否挽救边界失败工况。

- 检测器：`CenterDivergenceDetector`
- 响应：extend 段每步检测，滑动则手指握力 +0.015 rad，上限 0.25 rad

#### 阶段 C：方案二闭环验证

**目的**：验证竖直支撑力检测（含平滑、峰值 latch）的防滑效果。

- 检测器：`VerticalSupportAntislipDetector`
- 轨迹阶段预热 \(S_{avg}\)，extend 段闭环加力
- 信号曲线：`python3 scripts/plot_vertical_support_curves.py`

```bash
python3 scripts/run_ketchup_robustness_sweep.py --case friction_div2 --antislip
python3 scripts/render_friction_div2_compare.py    # 开环 vs 防滑对比视频
```

#### 判定标准

| 等级 | 条件 |
|------|------|
| **PASS** | 扩展段 Δz ≥ 6 cm，扩展接触 ≥ 50%（100/200 步），扩展段下落 ≤ 3 cm |
| **FAIL** | 失握、明显下落、或几乎无抬升 |

## 5. 结论总结

### 5.1 主要结论

1. **问题边界清晰**  
   在同一 SPIDER 开环轨迹下，我们既能稳定成功（质量 ×32），也能稳定失败（摩擦 ÷2~÷8），为防滑算法提供了可复现的对比基准。

2. **失败主因是摩擦不足，而非物体重量**  
   质量增至约 15.5 kg 仍可完成扩展段抬升；摩擦减半即导致扩展段中途滑落。

3. **方案一（掌系中心背离）验证通过**  
   在 friction÷2 上将扩展段从 **−20.7 cm 恢复为 +8.7 cm**，不依赖接触数变化，对齐真机触觉 + 物体位姿估计。

4. **方案二（竖直支撑力）验证通过**  
   达到与方案一相同的防滑效果（+8.7 cm，200/200 接触）。真机仅需接触力与重力方向；需 **200 ms 平滑**及**峰值 latch** 才能稳定闭环。

5. **两种方案互补**  
   方案一偏几何、触发更早；方案二纯力学、不依赖视觉。后续可融合为「几何主检测 + 力学确认」。

6. **极端工况仍具挑战**  
   friction÷4/÷8 在轨迹段已失握，仅在 extend 段介入无法挽救，需轨迹段提前防滑或更大握力响应能力。

### 5.2 工程要点

| 要点 | 说明 |
|------|------|
| 仿真为真机服务 | 不使用 MuJoCo 内部滑移速度等不可测量量 |
| 先摸底、再闭环 | 开环边界 → 算法介入 → 量化改善 |
| 力信号需平滑 | 瞬时 S(t) 噪声大，200 ms 窗口是必要的 |
| 闭环需 latch | 纯 S/S_avg 存在「误恢复」，峰值跟踪 + latch 是方案二成功的关键 |

### 5.3 下一步工作

| 优先级 | 方向 |
|--------|------|
| P0 | 方案一 + 方案二融合，降低 nominal 工况误触发 |
| P1 | **神经网络防滑**：[方案一二神经网络防滑检测设计.md](方案一二神经网络防滑检测设计.md) |
| P2 | friction÷4：轨迹段提前介入，或提高握力上限/响应速度 |
| P3 | 点云质心加噪声，验证方案一对位姿误差的鲁棒性 |
| P4 | 补全能量流（接触点速度、完整 \(\tilde{P}^R\)），与当前方案对比 |
| P5 | XHAND taxel 映射，统一仿真/真机日志，推进真机部署 |

### 5.4 一句话总结

> 针对 XHAND 番茄酱瓶抓取场景，我们提出了**掌系几何背离**与**竖直支撑力**两条真机可部署的防滑检测方案，并在仿真中实现了从开环 FAIL 到闭环 PASS 的完整验证；其中 friction÷2 边界工况下，两种方案均将扩展段抬升从 **−20.7 cm 恢复为 +8.7 cm**，为后续真机试验与算法融合奠定了基础。
