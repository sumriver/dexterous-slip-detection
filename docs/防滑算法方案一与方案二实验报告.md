# 灵巧手抓握防滑算法：方案一与方案二实验报告

**日期**：2026-07-06  
**项目**：dexterous-slip-detection  
**分支**：`main`  
**仓库**：https://github.com/sumriver/dexterous-slip-detection  

---

## 摘要

本报告系统整理番茄酱瓶抓取场景中，两种防滑检测方案（**方案一：接触中心背离**；**方案二：竖直支撑力**）的算法设计、信号特性与闭环实验结果。

在统一的开环 SPIDER 轨迹与扩展段（2 s 模仿 + 腕部 tz +10 cm）下：

| 工况 | 开环 | 方案一闭环 | 方案二闭环 |
|------|------|-----------|-----------|
| baseline（μ×1.0） | PASS | PASS | PASS |
| **friction÷2（μ×0.5）** | **FAIL** | **PASS** | **PASS** |
| friction÷4（μ×0.25） | FAIL | FAIL | FAIL |
| friction÷8（μ×0.125） | FAIL | FAIL | FAIL |

**核心结论**：方案一与方案二均能在 **friction÷2** 这一「可救边界工况」上将扩展段从 **−20.7 cm 恢复为 +8.7 cm**，实现完整防滑闭环验证；二者对更极端的低摩擦（÷4、÷8）均无法挽救。

---

## 1. 实验背景

### 1.1 研究目标

在灵巧手抓取与搬运过程中，利用**真机可观测信号**实时检测物体相对滑动，并在检测到滑动时**增大手指握力**，抑制滑落。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 仿真为真机服务 | 不使用 MuJoCo 内部「作弊」量（如精确接触滑移速度）；输入对齐 XHAND 可测信号 |
| 先摸底、再闭环 | 同一开环轨迹下，先确定成功/失败边界，再接入防滑算法 |
| 物理可解释 | 优先采用可解释的标量力学/几何指标，能量流 m̃ 完整版留作后续迭代 |

### 1.3 实验平台

| 项目 | 配置 |
|------|------|
| 灵巧手 | XHAND（SPIDER arcticv2 右手 workspace） |
| 物体 | 番茄酱瓶 `s01-ketchup_use_01`，质量约 **0.48 kg** |
| 仿真器 | MuJoCo，CPU，`sim_dt = 0.01 s` |
| 轨迹 | SPIDER 预优化轨迹回放 + **2 s 扩展段**（末 1 s 控制循环模仿 + 腕 tz 抬升 10 cm） |
| 场景构建 | `python3 scripts/build_spider_ketchup_right.py` |

### 1.4 判定标准

| 等级 | 条件 |
|------|------|
| **PASS** | 扩展段 Δz ≥ 6 cm，且扩展接触步数 ≥ 50%（100/200 步），扩展段物体下落 ≤ 3 cm |
| **PARTIAL** | 有抬升或部分接触，未达 PASS |
| **FAIL** | 扩展段失握、物体下落 > 3 cm、或几乎无抬升 |

### 1.5 统一防滑响应策略

两种方案共用同一握力控制器（`src/sim/antislip_control.py`），仅在 **extend 段**启用：

| 参数 | 数值 |
|------|------|
| 手指 actuator 索引 | 6–17（XHAND 右手） |
| 每步握力增量 | +0.015 rad |
| 累计上限 | 0.25 rad |
| 衰减 | 无（触发后保持至 extend 结束） |

## 2. 开环摸底（公共基线）

在接入任何防滑算法前，对物体物理参数进行扫描（`scripts/run_ketchup_robustness_sweep.py`）。

### 2.1 质量组（全部 PASS）

| Case | 质量 (kg) | 全程 Δz | 扩展段 Δz | 扩展接触 |
|------|-----------|---------|-----------|----------|
| baseline | 0.48 | +16.0 cm | +9.0 cm | 200/200 |
| mass×2 | 0.97 | +15.6 cm | +9.1 cm | 200/200 |
| mass×4 | 1.93 | +13.8 cm | +8.8 cm | 200/200 |
| mass×8 | 3.86 | +11.8 cm | +9.7 cm | 200/200 |
| mass×16 | 7.73 | +7.6 cm | +9.2 cm | 200/200 |
| mass×32 | 15.45 | +2.0 cm | +9.7 cm | 200/200 |

**结论**：加重不是开环失败主因；质量增至约 **15.5 kg** 仍可完成扩展段抬升。

### 2.2 摩擦组（核心失败工况）

| Case | 摩擦 × | 全程 Δz | 扩展段 Δz | 扩展接触 | 失败模式 |
|------|--------|---------|-----------|----------|----------|
| baseline | 1.0 | +16.0 cm | +9.0 cm | 200/200 | — |
| **friction÷2** | **0.5** | **−18.7 cm** | **−20.7 cm** | **99/200** | 扩展段中途滑落 |
| friction÷4 | 0.25 | −18.7 cm | ~0 cm | 0/200 | 轨迹末已失握 |
| friction÷8 | 0.125 | −14.0 cm | ~0 cm | 0/200 | 同上 |

**结论**：**低摩擦是主要失败因素**；**friction÷2** 为「部分失败」，最适合作为防滑闭环的首个验证工况。

## 3. 方案一：接触中心 vs 物体中心（掌系背离）

### 3.1 算法原理

在手掌坐标系（`right_hand_link`）下，比较**力加权接触中心**与**物体中心**的相对运动。不依赖接触数变化，可检测「接触数不变但物体在滑」的情形。

| 量 | 仿真实现 | 真机对应 |
|----|----------|----------|
| 接触中心 | `Σ\|F_i\|·p_i / Σ\|F_i\|` | 触觉 taxel 力 + 位置 |
| 物体中心 | body `xpos`（点云质心代理） | 点云 / 视觉 COM 估计 |
| 参考系 | `right_hand_link` 掌系 | 关节 FK 掌系 |

**判据**（extend 段内，自 extend 起点起）：

1. 记录基准间距 `sep_ref = ‖x_obj^H − x_contact^H‖`
2. 若 `|sep − sep_ref| ≥ 8 mm` → 滑动
3. 或两中心运动速度差 `≥ 15 mm/s` → 滑动

### 3.2 代码实现

| 文件 | 职责 |
|------|------|
| `src/sim/slip_center_detect.py` | `CenterDivergenceDetector` 掌系背离检测 |
| `src/sim/antislip_control.py` | `GripBoostController` 握力递增 |
| `src/sim/spider_replay.py` | extend 段闭环（历史版本，`antislip_scheme=1`） |

### 3.3 信号特性（friction÷2，extend 段）

| 指标 | 数值 |
|------|------|
| 首次滑动检测 | extend 开始后 **~0.01 s**（t ≈ 3.02 s） |
| extend 全程滑动事件 | **199 / 200 步** |
| 最大握力增量 | **0.25 rad**（第 ~17 步达到上限） |
| 特点 | 从 extend 一开始就持续报警，响应极早 |

方案一对几何相对运动敏感，在 friction÷2 下几乎每步均判定为滑动，握力快速拉满。

### 3.4 闭环实验结果

| Case | 开环 | 方案一 | 扩展段 Δz | 扩展接触 | 滑动事件 | 最大握力 |
|------|------|--------|-----------|----------|----------|----------|
| baseline | PASS | **PASS** | +7.4 cm | 200/200 | 199 | 0.25 rad |
| **friction÷2** | **FAIL** | **PASS** | **+8.7 cm** | **200/200** | **199** | **0.25 rad** |
| friction÷4 | FAIL | FAIL | ~0 cm | 200/200 | 0 | 0.00 rad |
| friction÷8 | FAIL | FAIL | ~0 cm | 200/200 | 0 | 0.00 rad |

**friction÷2 前后对比**：

```
开环：  扩展段 Δz = −20.7 cm，接触 99/200，物体坠落
方案一：扩展段 Δz = +8.7 cm，接触 200/200，握力顶满 0.25 rad
```

**friction÷4/÷8 说明**：轨迹段结束时物体已几乎失握，extend 段虽可能有接触（÷4 时 200/200），但无法产生有效抬升；方案一在 extend 段未检测到几何背离（滑动事件 0），故未触发握力补偿。

### 3.5 方案一评价

| 维度 | 评价 |
|------|------|
| 优点 | 检测极早（extend 起始即触发）；不依赖接触数；对齐真机触觉 + 视觉/点云 |
| 缺点 | friction÷2 下几乎每步报警（高敏感）；baseline 也持续触发（199/200），依赖握力上限不破坏 nominal 抓取 |
| 真机适配性 | 高——仅需触觉力位置 + 物体位姿估计 |

## 4. 方案二：竖直支撑力 S(t)

### 4.1 算法原理

利用手–物接触力在**竖直方向（反重力 ĝ）**的投影，度量握持对物体的「支撑能力」。

**瞬时支撑力**：

$$S(t) = \sum_{i \in \text{hand–object}} \max\!\big(0,\; \mathbf{f}_i \cdot \hat{\mathbf{g}}\big)$$

其中 $\mathbf{f}_i$ 为作用在物体上的接触力（世界系），$\hat{\mathbf{g}}$ 为反重力单位向量。

**信号处理管线**（观测用 + 闭环用）：

```
S_raw  →  S_smooth（200 ms 滑动平均）  →  S_avg（2 s 滑动平均，基于 S_smooth）
```

**滑动判据**（`VerticalSupportAntislipDetector`）：

| 条件 | 说明 | 默认阈值 |
|------|------|----------|
| 比值判据 | `S_smooth / S_avg < τ` | τ = 0.7 |
| 峰值判据 | `S_smooth < ρ × peak(S_smooth)`（extend 段内峰值跟踪） | ρ = 0.95 |
| Latch | 任一条件触发后，整个 extend 段保持「滑动激活」 | — |

> **关键工程发现**：仅用 `S_smooth/S_avg` 而不加峰值跟踪与 latch，会在 friction÷2 下失败——力短暂回升导致误判「未滑移」，握力只加到 0.105 rad 即停止，物体随后坠落。加入峰值跟踪 + latch 后，检测时刻与方案一对齐（extend 开始后 ~0.02 s）。

轨迹阶段持续更新 `S_avg`（不重置），为 extend 段提供稳定基线。

### 4.2 代码实现

| 文件 | 职责 |
|------|------|
| `src/sim/slip_vertical_support.py` | `measure_vertical_support`、`VerticalSupportMovingAverage`、`VerticalSupportAntislipDetector` |
| `src/sim/antislip_control.py` | 共用握力控制器 |
| `src/sim/spider_replay.py` | extend 段闭环（当前默认 `antislip_scheme=2`） |
| `scripts/plot_vertical_support_curves.py` | 方案二信号曲线绘制 |

### 4.5 方案二评价

| 维度 | 评价 |
|------|------|
| 优点 | 纯力学信号，真机只需接触力 + 重力方向；比值+峰值双判据物理含义清晰 |
| 缺点 | 原始 S(t) 噪声大，需 200 ms 平滑；纯 S/S_avg 有「误恢复」问题，必须加峰值 latch |
| 真机适配性 | 高——仅需六维力/触觉合力与 IMU/重力方向 |

## 5. 方案一 vs 方案二 对比

### 5.1 核心指标对照（friction÷2）

| 指标 | 开环 | 方案一 | 方案二 |
|------|------|--------|--------|
| 扩展段 Δz | −20.7 cm | **+8.7 cm** | **+8.7 cm** |
| 扩展接触 | 99/200 | **200/200** | **200/200** |
| 判定 | FAIL | **PASS** | **PASS** |
| 首次检测时刻 | — | ~3.02 s | ~3.03 s |
| extend 滑动事件 | — | 199 | 194 |
| 最大握力 | 0 | 0.25 rad | 0.25 rad |

### 5.2 检测机理对比

| 维度 | 方案一（几何背离） | 方案二（竖直支撑力） |
|------|-------------------|---------------------|
| 核心信号 | 接触中心 vs 物体中心间距/速度差 | S_smooth、S_avg、extend 峰值 |
| 物理含义 | 物体相对手滑移 | 握持支撑力塌缩 |
| 检测时机 | extend 起始即敏感 | 需峰值 latch 才达同等灵敏度 |
| 噪声特性 | 几何量相对稳定 | S_raw 高频抖动，需 200 ms 平滑 |
| 误报（baseline） | 199/200 步触发（高敏感） | 195/200 步触发（峰值判据） |
| 实现复杂度 | 中等（需物体位姿） | 中等（需力分解 + 双窗口） |

### 5.3 适用场景建议

| 场景 | 推荐 |
|------|------|
| 有可靠物体位姿（视觉/点云） | 方案一：几何直观，响应最早 |
| 仅有触觉力、无视觉 | 方案二：不依赖物体位姿 |
| 融合方案 | 方案一主检测 + 方案二辅助确认，降低误报 |
| friction÷4 及更极端 | 需轨迹段提前介入或更大握力上限 |

## 8. 结论

1. **开环边界清晰**：质量 ×32 仍 PASS，摩擦 ÷2 为首个 FAIL，÷4/÷8 更严重。  
2. **方案一**（掌系中心背离）在 friction÷2 实现 **−20.7 cm → +8.7 cm** 的闭环恢复，检测极早，真机仅需触觉 + 物体位姿。  
3. **方案二**（竖直支撑力 S_smooth/S_avg + 峰值 latch）达到**与方案一相同的防滑效果**，真机仅需接触力 + 重力方向。  
4. 方案二原始 S(t) 噪声大，需 **200 ms 平滑**；纯比值判据存在误恢复，**峰值跟踪 + latch** 是闭环成功的关键。  
5. 两种方案对 friction÷4/÷8 均无效，需从轨迹段提前介入或提高握力响应能力。
