## 摘要

触觉传感器在原理（压阻 / 磁阻 / 光学）、形态（taxel 阵列 / 触觉图像 / 单点 F/T）和分辨率上差异极大，导致跨平台算法难以直接复用。行业普遍采用**分层抽象 + 约定 schema + 可选 latent 对齐**三条路径：工程侧用 ROS / LeRobot 等统一消息格式；学习侧用 T3、TactX、UniForce 等将异构输入映射到共享表征空间；控制侧则将高维传感降维为接触力学标量（如垂直支撑力 S）驱动防滑决策。

本调研回答三个问题：（1）算法上如何抽象并统一触觉 I/O；（2）行业如何抽象；（3）本体真机数据采集主要采哪些触觉相关数据。结论对 XHAND 防滑研究与 NN-0 数据集设计具有直接参考价值。

## 1. 触觉传感器差异大，算法上如何抽象并统一 I/O？

核心思路不是「把所有传感器变成同一种硬件」，而是**分层抽象 + 约定 schema + 可选 latent 对齐**。

### 1.1 分层抽象（从低到高）

行业里普遍采用 4～5 层，上层对下层可降维，下层对上层可升维：

```
L0 原始传感层    taxel / 触觉图像 / 磁阻 / 压阻
      ↓
L1 物理量层      Fn, Ft, 变形深度, 温度
      ↓
L2 接触几何层    接触中心 COP, 接触 patch, 法向
      ↓
L3 接触力学层    每指 6D wrench, 总接触力
      ↓
L4 任务标量层    S, slip score, grasp stability
```

| 层级 | 典型内容 | 适用算法 |
|------|----------|----------|
| L0 原始 | taxel 阵列 `(H×W×C)`、GelSight 图像、FlexiTac 变形图 | 端到端 NN、表征学习 |
| L1 物理量 | 法向/切向力、变形深度、温度 | 规则防滑、物理先验 |
| L2 几何 | 接触中心、接触面积、滑动方向 | 方案一 contact-center divergence |
| L3 力学 | 每指 `fx,fy,fz,tx,ty,tz` | 阻抗控制、力闭环 |
| L4 任务 | 垂直支撑力 S、滑移概率 | 方案二 antislip、轻量部署 |

### 1.2 算法侧统一 I/O 的推荐做法

#### （1）输入侧：Driver Adapter + Canonical Frame

- 每个传感器实现 `read() -> TactileObservation`
- 统一坐标系：指尖局部系（法向为 z，切向 x/y）
- 统一时间戳、采样率、偏置校正（tare）
- 缺失通道用 mask 标记，而不是硬填 0

#### （2）中间表示：Contact Patch 或 Wrench

最实用的「算法 lingua franca」是 **L3 接触力学层**：

- ROS `tactile_toolbox`：`TactileState`（原始 taxel）→ `TactileContacts`（每 link 一个 wrench + 接触位置/法向）
- 下游防滑/抓取算法只依赖 L3，不绑具体 taxel 布局

#### （3）学习侧：Sensor-specific Encoder → Shared Latent

当传感器形态差异太大（压阻 vs 光学 vs 磁阻），不在输入格式上强行统一，而在 **latent space** 对齐：

| 方法 | 机制 |
|------|------|
| **T3 / FoTa** | sensor encoder + shared transformer trunk + task decoder |
| **TactX** | 每传感器 encoder/decoder，16D 共享 latent，对比学习 + 交叉重建 |
| **UniForce** | CVAE，逆动力学（图像→力 latent）+ 正动力学（力→图像），静力平衡跨传感器标定 |

#### （4）输出侧：按任务定义，而非按传感器

- **防滑**：`slip_detected: bool` + `confidence` + 可选 `trigger_features`（如 S/S_avg）
- **力控**：`target_wrench` 或 `Δgrasp_force`
- **模仿学习**：`observation.tactile` 向量或 token

### 1.3 对 XHAND 的直接映射

**硬件规格**（XHAND1）：

- 每指 12×10 taxel，每点 Fn + Ft_x + Ft_y（共 360 维/指）
- 5 指合计 600 个三维力测点
- 另有温度阵列；关节位置、速度、电流（力矩）

**与本项目的对应关系**：

| 抽象层 | 本项目实现 |
|--------|------------|
| L0→L1 | `TactileFrame: (5, 12, 10)` normal + tangential（`xhand_tactile_sim.py`） |
| L2 | 方案一：contact center vs object center divergence |
| L4 | 方案二：垂直支撑力 S = Σ Fn，S_smooth/S_avg 防滑触发 |

### 1.4 建议的统一 Schema（示意）

```python
TactileObservation = {
    "timestamp": float,
    "frames": [  # 每指一条
        {
            "link": "thumb_tip",
            "taxel_grid": (12, 10, 3),  # fn, ft_x, ft_y
            "wrench_6d": (6,),         # 由 taxel 聚合，可选
            "temperature": (12, 10),     # 可选
            "valid_mask": (12, 10),
        },
        # ... 其余四指
    ],
    "task_scalars": {  # 算法层输出/中间量
        "vertical_support_S": float,
        "slip_score": float,
    }
}
```

## 2. 行业是如何抽象的？

可归纳为 **三条主线**，往往组合使用。

### 2.1 主线 A：消息/数据格式标准化（工程栈）

| 体系 | 抽象方式 | 典型字段 |
|------|----------|----------|
| **ROS tactile_toolbox** | 原始 → 接触聚合 | `TactileState` → `TactileContacts`（wrench + 位置 + 法向） |
| **LeRobot / LeFlexiTac** | 在现有 `observation.*` 上扩展触觉流 | `observation.tactile`（float32 向量）+ 可选 `observation.images.tactile_*` |
| **Sharpa RoCo / Origami** | 双轨：紧凑向量 + 高分辨率视频 | `observation.tactile` 60D（10 指 × 6 轴）+ `tactile_deform` / `tactile_raw` 视频 |
| **T-Rex** | 每指三模态并行 | raw 触觉图 + deformation map + 6 轴 F/T |
| **FoTa (T3)** | 多数据集聚合为统一训练格式 | 13 种传感器、11 种任务，统一 pretrain schema |

**LeRobot 生态**的做法最贴近「可落地」：不改整条 pipeline，只加一条 `observation.tactile.*`，策略侧用少量 token/conditioning 接入。LeFlexiTac 在 SO-100/101 上验证，同一触觉流可提升 ACT、Diffusion Policy、Pi0.5、SmolVLA 四类策略的成功率。

**Sharpa RoCo 数据集**的触觉字段定义（LeRobot 3.0）：

| 字段 | 类型 | Shape | 说明 |
|------|------|-------|------|
| `observation.tactile` | float32 | 60 | 10 指尖 × 6 轴（fx,fy,fz,tx,ty,tz） |
| `observation.images.tactile_deform` | video | 480×1200×3 | 变形导向触觉视频流 |
| `observation.images.tactile_raw` | video | 480×1600×3 | 原始触觉相机流 |

### 2.2 主线 B：表征学习（学术 / 跨传感器）

| 方法 | 核心思想 | 跨传感器能力 |
|------|----------|--------------|
| **T3 + FoTa** | Shared trunk + per-sensor encoder | 主要在 vision-based 触觉族内；FoTa 含 300 万+ 数据点、13 传感器 |
| **TactX** | 16D latent，resistive/magnetic/vision 三模态 | 零样本跨传感器策略迁移约 28%→46% |
| **UniForce** | 力 grounded latent，传感器互压采集配对数据 | GelSight/TacTip/uSkin 零样本力估计 |
| **AnyTouch / Sparsh** | 大规模自监督预训练触觉 encoder | 偏 vision-tactile |

趋势：**输入格式可以异构，策略消费的是 latent 或 wrench，而不是原始 taxel 布局。**

### 2.3 主线 C：任务驱动降维（控制 / 防滑）

与本项目最接近的路径：

- 不把 600× taxel 直接喂给控制器
- 先聚合为**接触力学标量**（总法向力、支撑力 S、切向力变化率）
- 再触发**离散事件**（滑移检测 → 增握力）

这是工业部署里最常见、也最稳的路径：**高维传感，低维决策。**

### 2.4 ROS tactile_toolbox 抽象链路

[tactile_toolbox](https://github.com/ubi-agni/tactile_toolbox) 是 ROS 侧较完整的触觉标准化方案：

```
原始 taxel (TactileState)
    → contact_force_estimator
    → 每 link 聚合 wrench (TactileContacts)
    → tactile_pcl / rviz 可视化
```

关键能力：

- `tactile_bias`：按需 tare（去皮）
- `tactile_state_publisher`：多源合并
- `contact_force_estimator`：taxel → 接触位置 + 力 + 法向

## 3. 本体真机数据采集：触觉方面主要采什么？

综合 T-Rex、Sharpa RoCo、HRDexDB、DexViTac、MILE、FreeTacMan、LeFlexiTac 等近期公开数据集与系统，真机采集的触觉相关模态可归纳如下。

### 3.1 几乎必采（dexterous hand 真机）

| 模态 | 典型频率 | 说明 |
|------|----------|------|
| **关节位置 q** | 30–100 Hz | 全手 DoF |
| **关节速度 q̇** | 同步 | 常与 q 同采 |
| **关节电流/力矩 τ** | 同步 | 无指尖 F/T 时常作接触力代理（QDD 标定） |
| **每指尖 6 轴 wrench** | 30 Hz | T-Rex、RoCo 均作为紧凑触觉主通道 |
| **多视角 RGB** | 30 Hz | 头相机 + 腕相机，与触觉时间对齐 |

### 3.2 高分辨率触觉（新一代 dexterous 平台标配）

| 模态 | 代表平台 | 用途 |
|------|----------|------|
| **原始触觉图像** | Sharpa Wave (T-Rex) | 240×320，接触形貌 |
| **变形图 deformation map** | Sharpa, RoCo | 强调接触引起的表面变化 |
| **Taxel 阵列 Fn/Ft** | XHAND 12×10×3 | 空间分辨的力分布，适合滑移/纹理 |
| **温度阵列** | XHAND | 材质识别、抓取状态辅助 |

### 3.3 标定与 Ground Truth（实验室级）

| 模态 | 用途 |
|------|------|
| **外置 F/T 传感器** | 标定指尖力估计、验证防滑阈值 |
| **物体 6D pose** | 滑移 ground truth（位姿变化） |
| **接触图 / COP** | HRDexDB 的距离基接触标注 |
| **成功/失败标签** | 抓取稳定性监督 |

### 3.4 同步与元数据

- **统一时钟**：视觉、关节、触觉同一 time base（T-Rex/RoCo 均为 30 Hz 对齐）
- **动作标签**：`state` + `action`（目标关节或 EF pose）
- **语言指令**（大规模 policy 数据集）：T-Rex 每 episode 有 VLM 生成描述
- **偏置校正**：采集前 tare，抓取前 warm-up 段（与方案二 S_avg 2s 思路一致）

### 3.5 代表性数据集触觉模态对照

| 数据集 | 触觉模态 | 规模 |
|--------|----------|------|
| **T-Rex** | 10 指 ×（240×320 触觉图 + 240×240 变形图 + 6 轴 F/T） | 50h，200+ 物体 |
| **Sharpa RoCo** | 60D wrench + 触觉 deform/raw 视频 | 双臂 + 双手 |
| **HRDexDB** | 指尖触觉 F^tactile + 多视角 RGB + 关节状态 | 1.4K trials，100 物体 |
| **DexViTac** | 高密度触觉 + 第一视角视觉 + 手部运动学 | 2400+ demos |
| **MILE** | 指尖 visuotactile + RGB-D + 关节位置 | 等构外骨骼采集 |
| **FreeTacMan** | visuo-tactile 图像对 + OptiTrack 位姿 | 3000k+ 图像对 |
| **LeFlexiTac** | FlexiTac 变形图 → `observation.tactile` | SO-100/101 夹爪 |

### 3.6 T-Rex 单 Episode 数据结构（参考）

T-Rex 在 Sharpa Wave 手上每指同步三种触觉模态：

1. **原始触觉图像**：240×320
2. **变形深度图**：240×240
3. **6 轴净 wrench**：fx, fy, fz, tx, ty, tz

同步频率 30 Hz，与 3 路 RGB、关节 state/action、语言指令对齐。

### 3.7 按传感器类型的采集侧重

| 传感器类型 | 真机主要存什么 |
|------------|----------------|
| **Taxel 阵列（XHAND 类）** | 原始 grid + 聚合 wrench + 关节 τ |
| **Vision-based（GelSight/DIGIT/FlexiTac）** | 触觉图像 + 可选变形图 + 低维 summary 向量 |
| **仅力矩估计（无指尖传感）** | q, q̇, τ + 外置 F/T 标定片段 |

## 4. 对本项目防滑研究线的建议

结合已有仿真架构（方案一/二、NN-0~NN-4 路线图），真机数据采集建议对齐行业「双轨」做法：

### 4.1 原始轨（训练 NN-0~NN-4）

- 5×(12×10×3) taxel + 关节 (q, q̇, τ) + 物体位姿（若有）
- 标签：`slip_event`（由物体 6D 或外置 F/T 标注）+ 方案一/二的 S、center_div 作为 teacher

### 4.2 紧凑轨（部署 / 对比行业 baseline）

- 每指 6D wrench + 标量 S
- 导出为 LeRobot 风格 `observation.tactile`（30D 单臂手）

### 4.3 算法抽象优先级

| 阶段 | 路径 |
|------|------|
| 短期 | L1→L4（taxel → S → 防滑），与当前 Scheme 2 一致 |
| 中期 | 加 L2 contact center（Scheme 1）作为 NN 辅助特征 |
| 长期 | 若引入其他手/传感器，用 encoder 对齐 latent，不改防滑核心逻辑 |

### 4.4 与 NN 数据集设计的衔接

参见 [方案一二神经网络防滑检测设计](./方案一二神经网络防滑检测设计.md) 中 NN-0 导出规范。建议 HDF5 / LeRobot 字段：

```
observation.state          # 关节 q, q̇
observation.tactile        # taxel 展平或 per-finger wrench
observation.tactile_S      # 方案二 teacher 标量
observation.slip_label     # 0/1 滑移事件
action                     # 关节目标或 grasp force delta
```

## 6. 结论

1. **算法抽象**：采用分层表示（L0–L4），中间层以 per-finger wrench 或 contact patch 为通用接口；跨传感器差异用 encoder→latent 解决，而非强行统一原始格式。
2. **行业实践**：工程侧走 LeRobot/ROS schema 扩展；学习侧走 T3/TactX/UniForce 大规模预训练；控制侧走任务标量降维（与本项目 Scheme 2 一致）。
3. **真机采集**：关节本体感知 + 每指 6D wrench 是标配；新一代平台额外采集触觉图像/变形图；XHAND 应重点采集 12×10×3 taxel 原始阵列并同步关节 τ，辅以物体位姿或外置 F/T 作滑移标注。
