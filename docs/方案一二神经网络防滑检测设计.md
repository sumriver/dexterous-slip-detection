## 摘要

本文档规定将现有**方案一（掌系中心背离）**与**方案二（竖直支撑力）**转化为**神经网络防滑检测器**的正式技术路线。核心思路不是抛弃已验证的规则方案，而是将其作为**特征工程基线**与**教师标签（Teacher）**，在 MuJoCo 仿真中自动生成大规模标注数据，训练轻量时序网络，最终在 XHAND 真机上以 CPU 实时推理部署。

推荐实施路径：**知识蒸馏（NN-1）→ 特征时序网络（NN-2）→ 原始触觉端到端（NN-3，长期）**。

在 friction÷2 边界工况上，规则方案已将扩展段从 **−20.7 cm 恢复为 +8.7 cm**；神经网络阶段的验收标准为：**不低于规则方案闭环效果**，并在 baseline 工况上**降低误触发率**。

## 1. 背景与动机

### 1.1 现状

| 方案 | 检测机理 | friction÷2 闭环 | 主要局限 |
|------|----------|-----------------|----------|
| **方案一** | 接触中心 vs 物体中心掌系背离 | PASS（+8.7 cm） | extend 段几乎每步报警（高敏感）；依赖物体位姿 |
| **方案二** | 竖直支撑力 S_smooth/S_avg + 峰值 latch | PASS（+8.7 cm） | 需手工平滑窗口与 latch；S_raw 噪声大 |

二者均已接入统一握力响应（`GripBoostController`），仅在 extend 段闭环启用。

### 1.2 为何引入神经网络

| 动机 | 说明 |
|------|------|
| **自动融合** | 学习方案一（几何早预警）与方案二（力学确认）的最优组合，减少手工阈值 |
| **降噪** | 从含噪触觉序列中学习时序模式，替代固定 200 ms / 2 s 滑动窗口 |
| **可扩展** | 同一框架可吸纳更多特征（关节力矩、腕部加速度、物体速度估计误差等） |
| **真机适配** | 通过仿真数据增强（taxel 噪声、位姿误差）提升 sim-to-real 鲁棒性 |
| **为学习控制铺路** | 检测网络输出可作为 RL / 模型预测控制（MPC）的状态输入 |

### 1.3 设计约束（与规则方案一致）

1. **仿真为真机服务**：训练特征必须来自 XHAND 可测信号，不使用 MuJoCo 内部滑移速度作为部署输入（仅可作为训练标签）。
2. **CPU 实时**：`sim_dt = 10 ms`，推理延迟目标 **< 2 ms**（参数量 < 100K）。
3. **可回退**：部署时保留规则方案作为安全兜底（`--antislip-fallback=scheme2`）。
4. **当前无 GPU 训练环境**：首版使用 PyTorch CPU 或 ONNX；数据规模可控（单轨迹扩增至数千序列）。

## 2. 总体架构

### 2.1 系统框图

```
┌─────────────────────────────────────────────────────────────────┐
│                        传感层（真机 / 仿真）                      │
│  触觉力 F_i, p_i  │  关节 q, q̇  │  物体位姿（视觉/点云）│  ĝ   │
└────────────────────────────┬────────────────────────────────────┘
                             │
              ┌──────────────┴──────────────┐
              ▼                             ▼
     ┌─────────────────┐           ┌─────────────────┐
     │ 特征工程模块      │           │ 原始 taxel 栅格   │  ← NN-3 长期
     │ (方案一+二标量)   │           │ (12×10×3 / 指)   │
     └────────┬────────┘           └────────┬────────┘
              │ 窗口 T×D                       │
              ▼                                ▼
     ┌─────────────────────────────────────────────────┐
     │           时序神经网络（TCN / GRU / MLP）          │
     │   输出: p_slip ∈ [0,1]  可选: Δgrip* ∈ [0,0.25] │
     └────────────────────────┬────────────────────────┘
                              │ p_slip > τ
                              ▼
     ┌─────────────────────────────────────────────────┐
     │      GripBoostController（沿用现有握力响应）       │
     └─────────────────────────────────────────────────┘
```

### 2.2 三条技术路线

| 路线 | 名称 | 输入 | 标签 | 优先级 | 预期周期 |
|------|------|------|------|--------|----------|
| **A** | 知识蒸馏 | 方案一+二特征窗口 | 规则 slip 或仿真 GT | **P0** | 短 |
| **B** | 多任务学习 | 同上 | slip + 最优握力增量 | P1 | 中 |
| **C** | 端到端触觉 | 原始 taxel 时序 | 教师 / GT | P2 | 长 |

**本文档重点规定路线 A、B**；路线 C 仅给出接口预留。

## 3. 特征规范（路线 A / B）

每控制步 \(t\)（10 ms）构造特征向量 \(\mathbf{x}_t \in \mathbb{R}^D\)，建议 \(D \approx 32\text{–}48\)。

### 3.1 方案一特征（几何通道）— 8 维

| 索引 | 符号 | 定义 | 掌系 / 世界系 |
|------|------|------|--------------|
| 0 | `n_contacts` | 手–物接触数 | — |
| 1 | `sep` | \(\|x_{obj}^H - x_{contact}^H\|\) | 掌系 |
| 2 | `sep_delta` | \(|sep - sep_{ref}|\)（extend 内） | 掌系 |
| 3–5 | `v_div` | \(\|v_{obj}^H - v_{contact}^H\|\) 三分量 | 掌系 |
| 6 | `sep_rate` | \((sep_t - sep_{t-1}) / \Delta t\) | 掌系 |
| 7 | `contact_force_norm` | \(\sum_i \|F_i\|\) | 世界系 |

**仿真实现**：`CenterDivergenceDetector`（`src/sim/slip_center_detect.py`）  
**真机对应**：taxel 力加权位置 + 点云 COM + FK 掌系

### 3.2 方案二特征（力学通道）— 10 维

| 索引 | 符号 | 定义 |
|------|------|------|
| 8 | `S_raw` | \(\sum \max(0, F_i \cdot \hat{g})\) |
| 9 | `S_smooth` | 200 ms 滑动平均 |
| 10 | `S_avg` | 2 s 滑动平均（基于 S_smooth） |
| 11 | `S_ratio` | \(S_{smooth} / S_{avg}\) |
| 12 | `S_peak_ratio` | \(S_{smooth} / \max_{extend} S_{smooth}\) |
| 13 | `S_n` | 法向分量竖直投影之和 |
| 14 | `S_t` | 切向分量竖直投影之和 |
| 15 | `S_integral_0.5s` | 0.5 s 窗口积分 \(\int S\,dt\) |
| 16 | `mg_ratio` | \(S_{raw} / (mg)\) |
| 17 | `slip_rule_s2` | 方案二规则输出（0/1，训练时可作输入或仅作标签） |

**仿真实现**：`measure_vertical_support` + `VerticalSupportAntislipDetector`（`src/sim/slip_vertical_support.py`）

### 3.3 本体感知与上下文 — 8 维

| 索引 | 符号 | 定义 |
|------|------|------|
| 18–20 | `object_dz` | 物体 z 相对 extend 起点 / 相对轨迹末 |
| 21 | `wrist_tz` | 腕部 tz 控制量 |
| 22 | `wrist_tz_rate` | 腕部抬升速率 |
| 23 | `grip_extra` | 当前累计握力补偿 |
| 24 | `phase` | one-hot：trajectory / extend |
| 25 | `friction_scale` | 训练时工况编码（部署时可去掉或置 1） |

### 3.4 输入窗口

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 窗口长度 \(T\) | 30–50 步 | 300–500 ms 历史 |
| 采样率 | 100 Hz | 与 `sim_dt=0.01` 一致 |
| 归一化 | per-feature z-score | 在训练集上统计 \(\mu, \sigma\) |

网络输入张量形状：**`(batch, T, D)`**。

## 4. 标签策略

### 4.1 分类标签 \(y_{slip}\)

| 类型 | 定义 | 用途 |
|------|------|------|
| **教师 A（方案一）** | `CenterDivergenceDetector.slip` | 早预警、高召回 |
| **教师 B（方案二）** | `VerticalSupportAntislipDetector.slip_active` | 力学确认、含 latch |
| **融合教师** | \(y = y_1 \lor y_2\) | 默认蒸馏目标 |
| **仿真 GT** | \(\|v_{slip}\| > \epsilon_{slip}\)（物体相对掌系速度） | 仅训练用，不部署 |
| **事件标签** | 未来 \(H\) 步内物体 \(\Delta z < -\delta\) | 预测型防滑（可选） |

**推荐默认**：蒸馏目标为 **方案二 `slip_active`**（与当前 `--antislip` 默认一致）；对比实验增加方案一作教师。

### 4.2 回归标签 \(\Delta grip^*\)（路线 B，可选）

| 来源 | 定义 |
|------|------|
| 模仿学习 | 规则版在成功轨迹上每步 `GripBoostController.grip_extra` 增量 |
| 事后最优 | 动态规划：已知全轨迹时的最小握力序列（仿真 only） |

### 4.3 样本权重

对 friction÷2 / ÷4 失败片段**过采样**；对 extend 起点前后 50 步**加权**（滑移高发区）。

## 5. 模型结构

### 5.1 推荐基线：1D-TCN + MLP 头

```
Input (B, T, D)
  → Conv1d(D→64, k=3) + ReLU
  → Conv1d(64→64, k=3, dilation=2) + ReLU
  → GlobalAvgPool over T
  → FC(64→32) + ReLU
  → FC(32→1) + Sigmoid   → p_slip
```

| 属性 | 目标值 |
|------|--------|
| 参数量 | < 50K |
| 推理（CPU, T=40, D=32） | < 1 ms |
| 输出 | \(p_{slip} \in [0, 1]\) |

**部署阈值**：\(\tau = 0.5\)（验证集上按 F1 调优）；触发后沿用现有 `on_slip()` 握力 ramp。

### 5.2 备选：GRU

```
Input (B, T, D) → GRU(hidden=64, layers=1) → 末步隐状态 → MLP → p_slip
```

适合强时序依赖；参数量略大，仍满足 CPU 实时。

### 5.3 多任务头（路线 B）

```
共享骨干 →  p_slip (BCE)
          →  Δgrip* (MSE, 可选)
```

损失函数：

\[
\mathcal{L} = \mathrm{BCE}(p_{slip}, y_{teacher}) + \lambda \cdot \mathrm{MSE}(\widehat{\Delta grip}, \Delta grip^*)
\]

建议 \(\lambda = 0.1\)，以分类为主。

### 5.4 路线 C 预留：触觉 CNN-GRU

| 项目 | 规格 |
|------|------|
| 输入 | 每指 12×10 法向 + 切向力，或展平为向量 |
| 骨干 | 2D CNN（指内）→ 拼接 → GRU |
| 标签 | 同 4.1 |
| 依赖 | XHAND taxel 映射表（Phase 4） |

## 6. 数据集构建

### 6.1 仿真数据来源

在现有 `spider_replay.py` 循环中增加 `SlipDatasetLogger`：

for step in trajectory + extend:
    features = build_feature_vector(model, data, ...)  # §3

    }

### 6.2 工况覆盖

| 维度 | 取值 | 目的 |
|------|------|------|
| 摩擦 | ×1.0, ×0.5, ×0.25, ×0.125 | 正 / 负样本 |
| 质量 | ×1, ×2, ×4, ×8, ×16, ×32 | 鲁棒性 |
| 开环 / 闭环 | 开环轨迹 + 规则防滑成功轨迹 | 对比动态 |

单条 SPIDER 轨迹 × 10 摩擦质量组合 ≈ **数千至万级** extend 窗口样本（可扩）。

### 6.3 数据增强（训练时）

| 增强 | 参数 | 模拟 |
|------|------|------|
| 力噪声 | \(\mathcal{N}(0, \sigma_F)\)，\(\sigma_F = 0.05 \bar{F}\) | taxel 噪声 |
| 位姿抖动 | COM 偏移 ±3 mm | 点云误差 |
| 时间抖动 | 随机丢帧 5% | 通信丢包 |
| 窗口裁剪 | 随机起点 | 时序不变性 |

### 6.4 存储格式

## 7. 训练与验证协议

### 7.1 划分原则

- **按工况划分**：friction÷2 全部在 test 中保留一条轨迹；其余用于 train/val。
- **禁止泄漏**：同一仿真 seed 的 trajectory + extend 不跨集拆分。

### 7.2 指标

| 类型 | 指标 | 目标（相对规则方案二） |
|------|------|------------------------|
| 分类 | Precision / Recall / F1 | F1 ≥ 规则版 |
| 时序 | 检测提前量 \(\Delta t_{lead}\) | ≥ 规则版（friction÷2 上 ≥ 0.02 s） |
| 闭环 | extend Δz, 接触步数 | friction÷2：**≥ +8.7 cm, 200/200** |
| 误报 | baseline extend 误触发率 | **低于** 规则版（195/200 → 目标 < 50%） |
| 延迟 | 推理 + 特征计算 | < 2 ms / 步 |

### 7.3 训练超参（初版）

| 参数 | 值 |
|------|-----|
| Optimizer | Adam, lr=1e-3 |
| Batch size | 64 |
| Epochs | 50（早停 patience=8） |
| 类不平衡 | `pos_weight` 或 focal loss \(\gamma=2\) |

### 7.4 闭环评估

在 `replay_spider_task` 中新增 `--antislip-nn`：

```bash
python3 scripts/run_ketchup_robustness_sweep.py --case friction_div2 --antislip-nn
```

与 `--antislip`（规则方案二）及方案一并跑对比表。

## 8. 部署方案

### 8.1 推理管线（真机 / 仿真）

```
每 10 ms:
  1. 读取触觉 + 本体 + 位姿 → 更新特征环形缓冲
  2. NN 前向 → p_slip
  3. if p_slip > τ: GripBoostController.on_slip()
  4. （可选）若 NN 置信度极低且规则方案二报警 → 强制 on_slip()（融合兜底）
```

### 8.2 模型交付物

| 文件 | 说明 |
|------|------|
| `models/slip_tcn_v1.onnx` | ONNX 导出，CPU EP |
| `models/slip_tcn_v1_norm.json` | \(\mu, \sigma\) 归一化参数 |
| `src/sim/slip_nn_detector.py` | 推理封装，接口对齐 `VerticalSupportAntislipDetector` |

### 8.3 接口草案

```python
class SlipNeuralDetector:
    def __init__(self, model_path: str, norm_path: str, *, threshold: float = 0.5): ...
    def reset_extend(self) -> None: ...
    def update(self, features: np.ndarray) -> SlipNnReading:
        """features: 当前步 D 维；内部维护 T 窗口。"""
```

### 8.4 安全兜底

| 模式 | 行为 |
|------|------|
| `nn_only` | 仅神经网络 |
| `nn_or_scheme2` | \(p_{slip} > \tau\) 或 方案二报警 → 加力（推荐上线初期） |
| `scheme2_fallback` | NN 失败 / 超时 → 切回规则方案二 |

## 9. 与行业路线的关系

| 行业路线 | 本方案定位 |
|----------|-----------|
| NAIST+Honda 能量流 | 特征中可扩展 \( \tilde{m} \)、\(P^A\) 等；NN 作为能量流状态的非线性估计器 |
| CNN-LSTM 触觉 | 路线 C 对齐；路线 A 为低数据量过渡 |
| 端到端 RL（DexManip） | NN 检测器可作为 RL 的 slip 状态；防滑响应仍用 GripBoost |
| 振动检测 | **不适用**（XHAND 无振动通道） |

## 11. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 教师有偏（方案一过高敏） | 蒸馏继承误报 | 默认方案二作教师；GT 标签混合训练 |
| 仿真 GT 与部署特征不一致 | sim-to-real 落差 | 增强噪声；路线 C 逐步替换特征 |
| friction÷4/÷8 样本少 | 极端工况失效 | 过采样 + 课程学习（先 ÷2 再 ÷4） |
| 仅 extend 段训练 | 轨迹段失握无法检测 | 扩展标签到全轨迹；轨迹段提前介入（P1） |
| 算力 | 真机嵌入式 | TCN < 50K 参数；ONNX CPU |
